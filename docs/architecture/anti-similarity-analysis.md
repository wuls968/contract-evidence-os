# Anti-Similarity Analysis

Contract-Evidence OS is not a coding agent, browser agent, workflow builder, or thin supervisor-worker template.

## Coding Agent Contrast

Coding is treated as one possible execution capability. The system identity lives in:

- contract compilation,
- evidence-bound reasoning,
- shadow verification,
- audit-native delivery, and
- governed evolution.

## Browser Agent Contrast

Computer use is a late-stage tool harness behind policy checks and approval gates. It is
not the primary reasoning substrate and never the system's architectural center.

## Workflow Builder Contrast

The main abstraction is a compiled contract lattice and a plan graph with validation gates,
fallback edges, and approval semantics. Users do not assemble the system from blocks.

## Generic Multi-Agent Contrast

Specialist roles operate through capability passports and shared governance surfaces. They
write to a common evidence graph and audit ledger instead of free-form message passing.

## Plain RAG Contrast

Memory is a matrix of episodic, semantic, procedural, and policy stores with lifecycle
promotion and contamination controls, not a single retrieval bucket.
