# Release 0.9.0

`0.9.0` is the first repository-wide reset where the public story matches the actual runtime.

## Headline changes

- operator api v1 is now documented and versioned
- AMOS exposes a unified memory kernel surface instead of milestone-era scattered public views
- software control fabric has stable manifests, macros, action receipts, replay diagnostics, recovery hints, and failure patterns
- maintenance now has a resident daemon-oriented execution path in addition to direct worker-cycle calls
- operator metrics now have persisted history plus Prometheus/Grafana deployment assets for long-running local use
- container packaging now defaults to a runnable `ceos-server` startup path and the release CI verifies real sdist/wheel builds

## Intended use

This release is optimized for a local-first, long-running operator workflow with optional remote governance surfaces. It is still deliberately conservative:

- AMOS remains source-grounded and pointer-based
- software control remains governed and approval-aware
- maintenance remains specialized to memory operations rather than becoming a second general scheduler
