# ADR-053: Memory Artifact Registry And File Purge

## Status

Accepted

## Context

AMOS had deeper purge semantics inside its durable memory tables, but it still lacked a typed record of filesystem artifacts generated by rebuilds and exports. That left file cleanup outside the governed memory lifecycle.

## Decision

AMOS now persists `MemoryArtifactRecord` entries for governed memory artifacts such as:

- exported delivery files
- rebuilt dashboard indexes
- rebuilt project-state indexes

Hard purge can target these artifact records and physically delete the referenced files.

## Consequences

- Artifact deletion is now explicit and auditable instead of implicit.
- Rebuild and purge semantics cover both durable records and selected filesystem outputs.
- External artifact coverage is still scoped to AMOS-managed files, not arbitrary external storage.
