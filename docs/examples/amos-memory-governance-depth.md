# AMOS Memory Governance Depth Example

## Scenario

A task scope accumulates:

- raw task episodes
- semantic facts
- evidence packs and dashboard items
- a suspicious procedural candidate suggesting approval bypass

## Learned Admission

After repeated quarantine-shaped lifecycle traces, the scope trains an admission learning state.
The next risky candidate:

- is scored with the same base heuristics
- receives a learned poison boost
- crosses the quarantine boundary without crossing the block boundary

## Selective Purge

An operator decides to clear derived memory state but retain durable evidence.
Selective purge removes:

- evidence packs
- dashboard items
- risky governance artifacts

while leaving:

- raw episodes
- durable semantic facts

## Cross-Scope Timeline

Related task scopes store the same user-level predicate over time.
AMOS reconstructs a cross-scope timeline that shows:

- which state held first
- where the scope boundary changed
- where the state itself changed

## Policy Mining

Repeated lifecycle traces can then propose a memory-policy candidate.
That candidate still must:

- pass evaluation
- survive canary
- be explicitly promoted

before it changes runtime governance behavior.
